CONTEXT

ClawSentinel's /scan endpoint returns raw findings with no composite signal.
Users have no quick way to assess overall risk. This ADR adds a DangerScore.

DECISION

Add computeDangerScore(summary: SkillScanSummary): DangerScore to
skill-scanner.ts. Add DangerScore and DangerTier types to skill-scanner-types.ts.

FORMULA

  BaseScore = (critical_count * 40) + (warn_count * 10) + (info_count * 2)
  CategoryScore = max(OWASP_MULTIPLIERS[category] for all findings)
  RawScore = BaseScore * CategoryScore
  DangerScore = min(100, RawScore)

OWASP CATEGORY MULTIPLIERS (reviewed by Gemini 20260312-057-GEMINI):
  data-exfiltration:   2.0x  — LLM06 + LLM02
  credential-theft:    2.0x  — total system compromise vector
  code-injection:      1.8x  — LLM07 Insecure Plugin Design
  sandbox-escape:      1.8x  — LLM07 Insecure Plugin Design
  prompt-injection:    1.8x  — LLM01 #1 OWASP threat; elevated from 1.6x
  supply-chain:        1.5x  — LLM05
  inter-agent-attack:  1.5x  — LLM08 Excessive Agency
  cost-bombing:        1.5x  — LLM04 Model DoS; explicitly added (was 1.0x default)
  obfuscation:         1.2x  — evasion tactic, not direct vulnerability
  (all others:         1.0x)

RISK TIERS:
  SAFE        0-19   — scan clean or info-only findings
  REVIEW      20-49  — warn-level findings, human review recommended
  DANGEROUS   50-79  — critical findings present, do not install without review
  BLOCK       80-100 — confirmed high-severity bypass vectors

ABSOLUTE CONSTRAINT

ClawSentinel NEVER automatically blocks an install at any tier including BLOCK.
ClawSentinel scores, tiers, and advises. The user decides.
This constraint is product-level and may only be changed by Shem explicitly.

IMPLEMENTATION

  computeDangerScore() is unimplemented — OPEN-018, cleared for implementation.
  scorePluginBehavior() already exists in sentinel-hardening.ts (0-100 scale).
  These are parallel functions — neither depends on the other.
